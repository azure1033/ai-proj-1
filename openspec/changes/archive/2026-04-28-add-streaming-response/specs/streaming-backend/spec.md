## ADDED Requirements

### Requirement: SSE Stream Endpoint
The system SHALL support a streaming endpoint at `/ask?stream=true` using Server-Sent Events.

#### Scenario: Streaming request returns SSE
- **WHEN** client sends `POST /ask?stream=true` with a query
- **THEN** the response SHALL be `text/event-stream`
- **AND** events SHALL include `step`, `step_done`, `token`, and `done` types

#### Scenario: Non-streaming request returns JSON
- **WHEN** client sends `POST /ask` without `stream` parameter
- **THEN** the response SHALL be JSON (existing behavior)
- **AND** the response SHALL include `response`, `steps`, `session_id`

---

### Requirement: Token-Level Streaming
The system SHALL stream response tokens incrementally as they are generated by the LLM.

#### Scenario: Tokens are streamed one by one
- **WHEN** the LLM generates a response
- **THEN** each token group SHALL be emitted as an `event:token` SSE event
- **AND** the frontend SHALL append tokens to the message display in real-time

---

### Requirement: Agent Step Streaming
The system SHALL emit Agent tool calls and results as SSE events before the response tokens.

#### Scenario: Tool call emitted as step event
- **WHEN** the Agent calls a tool (e.g., web_search)
- **THEN** the system SHALL emit `event:step` with tool name and input
- **AND** the frontend SHALL show "正在搜索..." with the tool name

#### Scenario: Tool result emitted as step_done event
- **WHEN** a tool returns a result
- **THEN** the system SHALL emit `event:step_done` with tool output summary
- **AND** the frontend SHALL update the step status to "✓ 完成"

#### Scenario: Direct answer emits no steps
- **WHEN** the Agent answers directly without calling tools
- **THEN** the system SHALL emit only `event:token` and `event:done`
- **AND** no `event:step` events SHALL be emitted

---

### Requirement: Stream Error Handling
The system SHALL handle streaming errors gracefully.

#### Scenario: Agent fails during streaming
- **WHEN** the Agent encounters an error mid-stream
- **THEN** the system SHALL emit `event:error` with an error message
- **AND** close the SSE connection cleanly
